üîò Understanding NULL Values

What happens when data is missing? Let's understand NULL!

What is NULL?

NULL is SQL's way of saying "I don't know" or "this is empty" or "no value here."

It's NOT the same as zero. It's NOT an empty text. It's the absence of any value at all.

Real-World Example

Imagine you're collecting employee information:

  • John's phone number: 555-1234
  • Sarah's phone number: NULL (she didn't provide it)
  • Mike's phone number: 555-5678

Sarah's phone number is NULL because we don't have it. It's not zero. It's not blank. We simply don't know it.

NULL vs Zero vs Empty

These are all different:

  • NULL - No value at all (unknown)
  • 0 - The number zero (we know it's zero)
  • '' (empty text) - Empty string (we know it's empty)

Example with bank accounts:

  • Balance is $0 - You have an account with zero dollars
  • Balance is NULL - We don't know your balance (maybe account not set up yet)

Finding NULL Values

You can't use = with NULL. This is WRONG: 

-- This doesn't work!
SELECT first_name, phone
FROM employees
WHERE phone = NULL;

Instead, use IS NULL: 

SELECT first_name, phone
FROM employees
WHERE phone IS NULL;

This finds all employees who don't have a phone number on file.

üí° Remember: Use IS NULL, not = NULL

Finding Non-NULL Values

Want to find employees WHO DO have a phone number? Use IS NOT NULL: 

SELECT first_name, phone
FROM employees
WHERE phone IS NOT NULL;

This shows only people with phone numbers.

Example 1: Incomplete Customer Data

SELECT first_name, last_name, email
FROM customers
WHERE email IS NULL;

Find customers who didn't provide an email address. Time to reach out and get that info!

Example 2: Products Without Descriptions

SELECT product_name, description
FROM products
WHERE description IS NULL;

Find products missing descriptions so you can add them.

Example 3: Employees Without Managers

SELECT first_name, last_name, manager_id
FROM employees
WHERE manager_id IS NULL;

This might find the CEO or company owners (people at the top with no manager).

NULL in Calculations

Here's something tricky: if you do math with NULL, you get NULL!

  • 5 + NULL = NULL
  • 10 * NULL = NULL
  • 100 - NULL = NULL

Think about it: if you don't know one of the numbers, how can you calculate the answer?

Sorting with NULL

When you sort data, NULL values usually appear either first or last, depending on your database: 

SELECT first_name, phone
FROM employees
ORDER BY phone;

The employees with NULL phone numbers will be grouped together at the beginning or end.

Real-World Scenarios

Scenario 1: New Customers

Find customers who haven't made their first purchase yet: 

SELECT first_name, last_name
FROM customers
WHERE total_spent IS NULL;

Scenario 2: Unassigned Orders

Find orders that haven't been assigned to a salesperson: 

SELECT order_id, total_amount
FROM orders
WHERE salesperson_id IS NULL;

Scenario 3: Products Needing Attention

Find products missing important info: 

SELECT product_name
FROM products
WHERE price IS NULL OR stock_qty IS NULL;

Common Mistakes with NULL

Mistake #1: Using = Instead of IS

‚ùå Wrong:

WHERE phone = NULL

‚úÖ Right:

WHERE phone IS NULL

Mistake #2: Using != Instead of IS NOT

‚ùå Wrong:

WHERE phone != NULL

‚úÖ Right:

WHERE phone IS NOT NULL

Mistake #3: Thinking NULL = 0

NULL and 0 are completely different!

  • 0 = "I know the value is zero"
  • NULL = "I don't know what the value is"

Combining NULL with Other Conditions

You can mix NULL checks with other conditions: 

SELECT first_name, last_name, department, phone
FROM employees
WHERE department = 'Sales' AND phone IS NULL;

This finds Sales employees without phone numbers on file. 

SELECT product_name, price, stock_qty
FROM products
WHERE price > 50 AND description IS NOT NULL;

This finds expensive products that DO have descriptions.

Why NULL Exists

NULL is useful because:

  • Sometimes you don't know a value yet
  • Sometimes a value doesn't apply (like a middle name when someone doesn't have one)
  • Sometimes data is optional

Without NULL, you'd have to use fake values like 0 or "UNKNOWN", which can cause confusion.

üéØ Pro Tip: When building forms or apps, think about which fields are required and which can be NULL. This helps keep your database clean!

What You Learned

  • NULL means "no value" or "unknown"
  • NULL is different from 0 or empty text
  • Use IS NULL to find NULL values
  • Use IS NOT NULL to find values that exist
  • Never use = NULL - it doesn't work!
  • Math with NULL gives NULL

Understanding NULL helps you handle missing or incomplete data properly!
