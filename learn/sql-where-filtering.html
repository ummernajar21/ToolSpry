üîç Filtering Data with WHERE

Stop getting all the data. Learn to find exactly what you need!

Why Do We Need WHERE?

Imagine you have a table with 1,000 employees. You only want to see people from the Sales department. Do you want to scroll through all 1,000? No way!

That's where WHERE comes in. It's like a filter. You tell SQL: "Give me only the rows that match this condition."

The Basic Pattern

SELECT columns
FROM table
WHERE condition;

The WHERE line says: "Only show rows where this is true."

Example 1: Find Employees in Sales

SELECT first_name, last_name, department
FROM employees
WHERE department = 'Sales';

What happens here:

  • SQL looks at every employee
  • Checks if their department is "Sales"
  • Only shows the ones that match

üí° Important: When comparing text, use single quotes like 'Sales'. Numbers don't need quotes.

Comparison Operators

These are the symbols you use to compare things:

  • = means "equals" (is exactly the same)
  • > means "greater than" (bigger)
  • < means "less than" (smaller)
  • >= means "greater than or equal to"
  • <= means "less than or equal to"
  • != or <> means "not equal" (different)

Example 2: Find High-Paid Employees

SELECT first_name, last_name, salary
FROM employees
WHERE salary > 70000;

This finds everyone making more than $70,000. Notice how we DON'T use quotes around the number!

Example 3: Find Cheap Products

SELECT product_name, price
FROM products
WHERE price < 50;

This shows products under $50. Perfect for budget shopping!

Multiple Conditions: AND

What if you want employees who are BOTH in Sales AND earn more than $60,000? Use AND: 

SELECT first_name, last_name, department, salary
FROM employees
WHERE department = 'Sales' AND salary > 60000;

With AND, BOTH conditions must be true. It's like saying "I want this AND that."

Multiple Conditions: OR

Want employees from Sales OR Marketing? Use OR: 

SELECT first_name, last_name, department
FROM employees
WHERE department = 'Sales' OR department = 'Marketing';

With OR, at least ONE condition must be true. It's like saying "I want this OR that or both."

Real-World Examples

Example: Active Customers Only

SELECT first_name, last_name, email
FROM customers
WHERE is_active = 1;

Shows only customers who are still active (1 means yes, 0 means no).

Example: Recent Orders

SELECT order_id, total_amount
FROM orders
WHERE total_amount > 500;

Shows only big orders over $500.

Example: Find Specific Country

SELECT first_name, last_name, country
FROM customers
WHERE country = 'USA';

Shows only customers from the USA.

Common Mistakes

Mistake #1: Forgetting Quotes

‚ùå Wrong:

WHERE department = Sales

‚úÖ Right:

WHERE department = 'Sales'

Text needs quotes! Numbers don't.

Mistake #2: Using = Instead of ==

Good news: In SQL, you use ONE equals sign = for comparison. If you know other programming languages that use ==, forget that here!

Mistake #3: Case Sensitivity

In most databases, 'Sales' and 'sales' are DIFFERENT!

Always check the exact spelling and capitalization in your data.

Combining AND and OR

You can mix them, but use parentheses to be clear: 

SELECT first_name, last_name, department, salary
FROM employees
WHERE (department = 'Sales' OR department = 'Marketing')
  AND salary > 60000;

This finds people in Sales or Marketing who ALSO earn over $60,000.

The parentheses say "check these two departments first, THEN check the salary."

üéØ Pro Tip: Start simple. Get one WHERE condition working first. Then add more conditions one at a time!

What You Learned

  • WHERE filters rows based on conditions
  • Use = to check if things are the same
  • Use > < for numbers
  • Text needs quotes: 'Sales'
  • Numbers don't need quotes: 70000
  • AND means both conditions must be true
  • OR means at least one must be true

Now you can find exactly the data you want instead of getting everything!
