What is SQL?

SQL (Structured Query Language) is the language used to talk to databases. Think of it as asking questions to a spreadsheet, but much more powerful. SQL lets you retrieve, filter, and analyze data stored in tables.

The SELECT Statement

SELECT is the most important SQL command. It tells the database what data you want to retrieve.

Basic Syntax

SELECT column1, column2
FROM table_name;

Let's break this down:

  • SELECT - Specifies what columns you want
  • column1, column2 - The specific columns to retrieve
  • FROM - Indicates which table to get data from
  • table_name - The name of the table

Example 1: Select Specific Columns

Get first name and salary from employees table:

SELECT first_name, salary
FROM employees;

üí° Tip: Column names are separated by commas. Don't put a comma after the last column!

Example 2: Select All Columns

Use the asterisk (*) to select all columns:

SELECT *
FROM employees;

The * means "everything" - it retrieves all columns from the table.

Understanding Tables

A database table is like a spreadsheet with rows and columns:

  • Columns - The fields (like first_name, salary, department)
  • Rows - Individual records (each employee is one row)

Common Mistakes

‚ùå Wrong:

SELECT first_name, last_name,
FROM employees;

‚úÖ Correct:

SELECT first_name, last_name
FROM employees;

Why? No comma after the last column name!

‚ùå Wrong:

SELECT first_name
employees;

‚úÖ Correct:

SELECT first_name
FROM employees;

Why? Always include the FROM keyword!

Practice Examples

Try these queries in the Practice page:

  1. Get all employee information: 
    SELECT * FROM employees;
  2. Get only names and emails: 
    SELECT first_name, last_name, email
FROM employees;
  3. Get customer names: 
    SELECT first_name, last_name
FROM customers;
  4. Get product information: 
    SELECT product_name, price
FROM products;

Key Takeaways

  • SELECT chooses which columns to display
  • FROM specifies which table to query
  • Use * to select all columns
  • Separate column names with commas
  • SQL statements end with a semicolon (;)
