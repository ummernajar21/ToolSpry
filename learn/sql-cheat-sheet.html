üìã SQL Cheat Sheet (Beginner)

Quick reference for essential SQL commands. Everything you need to get started!

üìö What's Included: This cheat sheet covers the core SQL commands for beginners - SELECT, WHERE, ORDER BY, LIMIT, and NULL handling. Perfect for your first SQL queries!

Basic SELECT

Get data from a table.

Get all columns:

SELECT * FROM table_name;

Get specific columns:

SELECT column1, column2 FROM table_name;

Get first 10 rows:

SELECT * FROM table_name LIMIT 10;

Get unique values (no duplicates):

SELECT DISTINCT column_name FROM table_name;

WHERE Clause

Filter rows based on conditions.

Basic filter:

SELECT * FROM employees
WHERE department = 'Sales';

Comparison operators:

= equals WHERE age = 25
> greater than WHERE salary > 50000
< less than WHERE price < 100
>= greater or equal WHERE age >= 18
<= less or equal WHERE stock <= 10
!= or <> not equal WHERE status != 'Active'

Multiple conditions with AND:

SELECT * FROM employees
WHERE department = 'Sales' AND salary > 50000;

Both conditions must be true.

Multiple conditions with OR:

SELECT * FROM employees
WHERE department = 'Sales' OR department = 'Marketing';

At least one condition must be true.

Combine AND and OR (use parentheses!):

SELECT * FROM employees
WHERE (department = 'Sales' OR department = 'Marketing')
  AND salary > 50000;

ORDER BY

Sort results.

Ascending order (A-Z, 0-9) [Default]:

SELECT * FROM employees
ORDER BY last_name;

Descending order (Z-A, 9-0):

SELECT * FROM employees
ORDER BY salary DESC;

Sort by multiple columns:

SELECT * FROM employees
ORDER BY department, salary DESC;

Sorts by department first, then by salary within each department.

LIMIT

Control how many rows to return.

Get first 10 rows:

SELECT * FROM products
LIMIT 10;

Get top 5 highest salaries:

SELECT * FROM employees
ORDER BY salary DESC
LIMIT 5;

üí° Pro Tip: Always use ORDER BY with LIMIT, otherwise you get random rows!

NULL Values

Handle missing or unknown data.

Find rows with NULL values:

SELECT * FROM customers
WHERE email IS NULL;

Find rows with values (not NULL):

SELECT * FROM customers
WHERE email IS NOT NULL;

‚ö†Ô∏è Never use: WHERE column = NULL

‚úÖ Always use: WHERE column IS NULL

LIKE

Pattern matching in text. Use % as wildcard (means "anything").

Starts with 'John':

SELECT * FROM customers
WHERE first_name LIKE 'John%';

Finds: John, Johnny, Johnson

Ends with 'son':

SELECT * FROM customers
WHERE last_name LIKE '%son';

Finds: Anderson, Johnson, Wilson

Contains 'tech':

SELECT * FROM products
WHERE product_name LIKE '%tech%';

Finds: Technology, Techno, Biotech

BETWEEN

Find values in a range (inclusive).

Price range:

SELECT * FROM products
WHERE price BETWEEN 50 AND 100;

Includes 50 and 100.

Date range:

SELECT * FROM orders
WHERE order_date BETWEEN '2024-01-01' AND '2024-12-31';

IN

Match any value in a list.

Multiple values:

SELECT * FROM employees
WHERE department IN ('Sales', 'Marketing', 'IT');

This is shorter than:

WHERE department = 'Sales'
   OR department = 'Marketing'
   OR department = 'IT';

Combining Everything

SQL commands must be in this order:

  1. 1. SELECT - What columns to show
  2. 2. FROM - Which table
  3. 3. WHERE - Filter rows
  4. 4. ORDER BY - Sort results
  5. 5. LIMIT - How many rows

Complete example:

SELECT first_name, last_name, salary
FROM employees
WHERE department = 'Sales' AND salary > 50000
ORDER BY salary DESC
LIMIT 10;

This finds the top 10 highest-paid Sales employees earning over $50,000.

Important Rules

Quotes & Numbers

  • ‚úÖ Text needs single quotes: 'Sales'
  • ‚úÖ Numbers don't need quotes: 50000
  • ‚úÖ Dates need quotes: '2024-01-01'
  • ‚ùå Don't use: "Sales" (double quotes)

Case Sensitivity

  • ‚úÖ SQL keywords: NOT case-sensitive (SELECT = select)
  • ‚ö†Ô∏è Data values: ARE case-sensitive ('Sales' ‚â† 'sales')
  • ‚úÖ Column names: Usually not case-sensitive (depends on database)

Common Patterns

Top 10 most expensive products:

SELECT product_name, price
FROM products
ORDER BY price DESC
LIMIT 10;

Customers without email:

SELECT first_name, last_name
FROM customers
WHERE email IS NULL;

Products under $50 in Electronics:

SELECT product_name, price
FROM products
WHERE category = 'Electronics' AND price < 50
ORDER BY price;

Employees hired in 2023:

SELECT first_name, last_name, hire_date
FROM employees
WHERE hire_date BETWEEN '2023-01-01' AND '2023-12-31'
ORDER BY hire_date;

Find all Johns (any spelling):

SELECT * FROM customers
WHERE first_name LIKE 'John%';

Quick Tips

  • ‚úÖ Always test with LIMIT 10 first
  • ‚úÖ Use SELECT * to explore data initially
  • ‚úÖ Use DISTINCT to see unique values
  • ‚úÖ Check spelling - one typo = no results!
  • ‚úÖ Use parentheses when mixing AND/OR
  • ‚úÖ Remember: IS NULL not = NULL
  • ‚úÖ ORDER BY comes before LIMIT

Troubleshooting Empty Results

Step 1: Check if table has data

SELECT * FROM table_name LIMIT 10;

Step 2: Remove WHERE to see all data

SELECT * FROM table_name;

Step 3: Check what values exist

SELECT DISTINCT column_name FROM table_name;

Step 4: Check spelling and capitalization!

Most empty results are caused by typos.

Comments in SQL

Single line comment:

-- This is a comment
SELECT * FROM employees;

Multiple line comment:

/* This is a
   multi-line comment */
SELECT * FROM employees;
