üìä Sorting with ORDER BY & LIMIT

Put your results in order and control how many rows you see!

Why Sort Your Results?

Imagine you get a list of 100 employees with their salaries. They're all mixed up. Who earns the most? Who earns the least? You'd have to read through everything!

With ORDER BY, you can sort them instantly. Highest to lowest. Lowest to highest. A to Z. Whatever you need!

ORDER BY Basics

The pattern is simple: 

SELECT columns
FROM table
ORDER BY column_name;

This sorts the results by that column. By default, it goes from small to big (or A to Z).

Example 1: Sort Names Alphabetically

SELECT first_name, last_name
FROM employees
ORDER BY last_name;

This shows all employees sorted by last name. Anderson comes before Smith!

Example 2: Sort by Salary

SELECT first_name, last_name, salary
FROM employees
ORDER BY salary;

This shows employees from lowest-paid to highest-paid.

Reverse the Order: DESC

Want highest-paid first? Add DESC (which means "descending" or "going down"): 

SELECT first_name, last_name, salary
FROM employees
ORDER BY salary DESC;

Now you see the top earners first!

üí° Remember:

  • ORDER BY column - Small to big (A to Z)
  • ORDER BY column DESC - Big to small (Z to A)

Example 3: Newest Products First

SELECT product_name, launch_date
FROM products
ORDER BY launch_date DESC;

This shows the newest products at the top!

LIMIT: Control How Many Rows

Sometimes you don't need ALL the results. Maybe you just want the top 5 or top 10.

That's where LIMIT comes in: 

SELECT columns
FROM table
ORDER BY column
LIMIT number;

Example 4: Top 5 Highest Salaries

SELECT first_name, last_name, salary
FROM employees
ORDER BY salary DESC
LIMIT 5;

This shows only the 5 highest-paid employees. Perfect!

Example 5: Top 10 Expensive Products

SELECT product_name, price
FROM products
ORDER BY price DESC
LIMIT 10;

Want to see your premium products? This is how!

Example 6: Top 3 Biggest Customers

SELECT first_name, last_name, total_spent
FROM customers
ORDER BY total_spent DESC
LIMIT 3;

Find your VIP customers who spent the most money!

Combining WHERE, ORDER BY, and LIMIT

You can use all three together! Here's the order they go in: 

SELECT columns
FROM table
WHERE condition
ORDER BY column
LIMIT number;

Always this order: WHERE, then ORDER BY, then LIMIT.

Example 7: Top 5 Sales Employees

SELECT first_name, last_name, salary
FROM employees
WHERE department = 'Sales'
ORDER BY salary DESC
LIMIT 5;

This finds:

  • Only Sales department (WHERE)
  • Sorted by salary, highest first (ORDER BY DESC)
  • Shows just the top 5 (LIMIT)

Example 8: Cheap Electronics

SELECT product_name, price
FROM products
WHERE category = 'Electronics' AND price < 100
ORDER BY price
LIMIT 10;

Find the 10 cheapest electronics under $100!

Sorting by Multiple Columns

You can sort by more than one thing: 

SELECT first_name, last_name, department, salary
FROM employees
ORDER BY department, salary DESC;

This sorts by department first (alphabetically), then by salary within each department.

Common Mistakes

Mistake #1: Wrong Order

‚ùå Wrong:

SELECT first_name
FROM employees
LIMIT 5
ORDER BY salary DESC;

‚úÖ Right:

SELECT first_name
FROM employees
ORDER BY salary DESC
LIMIT 5;

ORDER BY must come BEFORE LIMIT!

Mistake #2: Forgetting DESC

If you want highest first, don't forget DESC!

Without it, you'll get lowest first (which might not be what you want).

Real-World Use Cases

Top Performers

SELECT first_name, last_name, salary
FROM employees

LIMIT 10;

See your highest-paid employees.

Recent Customers

SELECT first_name, last_name, signup_date
FROM customers
ORDER BY signup_date DESC
LIMIT 20;

See who joined recently.

Best-Rated Products

SELECT product_name, rating
FROM products
ORDER BY rating DESC
LIMIT 5;

Find your top-rated items!

Oldest Orders

FROM orders
ORDER BY order_date

See your oldest unfulfilled orders.

What You Learned

  • ORDER BY sorts your results
  • Default order is A-Z or small to big
  • Add DESC to reverse (Z-A or big to small)
  • LIMIT controls how many rows you get
  • Order matters: WHERE ‚Üí ORDER BY ‚Üí LIMIT
  • Great for finding "top 10" or "best/worst" lists
